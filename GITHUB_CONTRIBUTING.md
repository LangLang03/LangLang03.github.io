# 神人百科 - 项目技术文档

## 1. 项目概述
神人百科是一个记录编程圈各类神人的网站，详细记录他们的言行、作品和影响。本项目支持AI创作和真实数据区分，具有响应式设计和主题切换功能。

## 2. 项目结构
```
web/
├── css/
│   └── styles.css          # 主样式文件，包含所有页面样式定义
├── js/
│   └── main.js             # 主JavaScript文件，实现所有交互功能
├── s/                      # 神人详细信息目录，每个神人一个子目录
│   └── webide/             # WebIDE开发者信息目录
│       ├── entries.json    # WebIDE专属术语表
│       ├── list.html       # WebIDE列表页，展示WebIDE的基本信息
│       ├── software.html   # WebIDE软件评价页面
│       ├── personal.html   # WebIDE个人评价页面
│       ├── photos.html     # WebIDE照片集页面
│       └── photos/         # WebIDE照片存储目录
├── res/                    # 资源文件目录，存放头像等图片
│   └── wbide.jpg           # WebIDE头像图片
├── entries.json            # 全局术语表，定义网站中使用的所有术语
├── info.json               # 神人信息数据，包含所有神人和章节信息
├── index.html              # 网站主页，展示神人列表和网站特色
└── README.md               # 项目基本说明文档
```

## 3. 文件详细说明

### 3.1 index.html
- **功能**：网站主页，展示神人列表和网站特色
- **结构**：
  - 头部：包含logo、搜索框和主题切换按钮
  - 侧边栏：导航菜单
  - 内容区：英雄区域、网站特色、神人列表、关于我们
- **关键逻辑**：
  - 动态加载神人列表：通过`fetch('info.json')`获取神人数据
  - 主题切换：通过修改`data-theme`属性实现
  - 搜索功能：通过`initSearch()`函数初始化
- **修改注意事项**：
  - 保持HTML结构完整性
  - 确保所有链接正确
  - 遵循现有样式规范

### 3.2 info.json
- **功能**：存储所有神人信息数据
- **结构**：
  ```json
  {
    "gods": [
      {
        "id": "webide",
        "name": "WebIDE开发者",
        "category": "编程圈神人",
        "description": "...",
        "keywords": [...],
        "path": "/s/webide",
        "index": 9.5,
        "avatar": "/res/wbide.jpg",
        "sections": [
          {
            "id": "software",
            "title": "软件评价",
            "description": "...",
            "path": "/s/webide/software.html",
            "type": "评价",
            "ai": true,
            "data_verified": true
          }
        ]
      }
    ]
  }
  ```
- **关键字段**：
  - `id`：神人的唯一标识符
  - `name`：神人的显示名称
  - `sections`：神人的章节数组
  - `ai`：布尔值，表示是否为AI创作
  - `data_verified`：布尔值，表示数据是否已验证
- **修改注意事项**：
  - 确保JSON格式正确
  - 每个神人/章节必须有唯一ID
  - `ai`和`data_verified`必须是布尔值

### 3.3 entries.json
- **功能**：全局术语表，定义网站中使用的术语
- **结构**：
  ```json
  {
    "entries": [
      {
        "term": "WebIDE",
        "definition": "...",
        "category": "项目"
      }
    ]
  }
  ```
- **关键字段**：
  - `term`：术语名称
  - `definition`：术语解释
  - `category`：术语分类
- **修改注意事项**：
  - 确保JSON格式正确
  - 每个术语必须有唯一名称

### 3.4 css/styles.css
- **功能**：定义网站的所有样式
- **结构**：
  - 颜色变量：定义亮色和暗色模式的颜色
  - 基础样式：全局样式设置
  - 组件样式：各个页面组件的样式
  - 响应式设计：适配不同屏幕尺寸
- **关键逻辑**：
  - 使用CSS变量实现主题切换
  - 采用Flexbox和Grid布局
  - 响应式设计适配移动端
- **修改注意事项**：
  - 遵循现有CSS变量命名规范
  - 确保样式兼容亮色和暗色模式
  - 使用相对单位

### 3.5 js/main.js
- **功能**：实现网站的所有交互功能
- **主要函数**：
  - `setCookie()`/`getCookie()`：处理cookie
  - `highlightTerms()`：动态高亮术语
  - `generateStatusBanner()`：生成状态横幅
  - `initSearch()`：初始化搜索功能
- **修改注意事项**：
  - 保持函数命名的一致性
  - 添加适当的注释
  - 确保代码兼容不同浏览器

### 3.6 s/ 目录
- **功能**：存放各个神人的详细信息
- **结构**：
  - 每个神人一个子目录
  - 子目录包含：
    - `entries.json`：专属术语表
    - `list.html`：列表页
    - 各个章节页面
    - `photos/`：照片目录
- **修改注意事项**：
  - 确保所有页面都调用`generateStatusBanner()`函数
  - 照片文件名保持一致
  - 术语表格式与全局术语表一致

## 5. 核心功能说明

### 5.1 术语高亮功能
- **功能**：自动高亮页面中的术语，并在悬停时显示解释
- **实现**：通过`highlightTerms()`函数实现，从entries.json加载术语表
- **触发**：页面加载时自动执行

### 5.2 主题切换功能
- **功能**：支持亮色模式和暗色模式切换
- **实现**：通过修改`data-theme`属性和CSS变量实现
- **存储**：使用cookie存储主题偏好

### 5.3 状态横幅生成
- **功能**：在每个条目页面顶部显示AI和数据验证状态
- **实现**：通过`generateStatusBanner()`函数实现
- **触发**：每个条目页面加载时调用

### 5.4 搜索功能
- **功能**：支持按关键词搜索神人和条目
- **实现**：通过`initSearch()`函数实现
- **触发**：页面加载时初始化，用户输入时执行

## 6. 数据规范

### 6.1 神人数据规范
- `id`：唯一标识符，使用小写字母和数字
- `name`：显示名称，简短明了
- `category`：分类，如"编程圈神人"
- `description`：简短描述，不超过100字
- `keywords`：关键词数组，3-5个关键词
- `path`：详细信息页路径，格式为`/s/{id}`
- `index`：神人指数，0-10之间的数字
- `avatar`：头像图片路径
- `sections`：章节数组，每个章节包含id、title、description、path、type、ai、data_verified字段

### 6.2 章节数据规范
- `id`：唯一标识符，使用小写字母和数字
- `title`：章节标题
- `description`：章节描述，不超过50字
- `path`：章节页面路径，格式为`/s/{god_id}/{section_id}.html`
- `type`：章节类型，如"评价"、"个人"、"照片"
- `ai`：布尔值，表示是否为AI创作
- `data_verified`：布尔值，表示数据是否已验证

### 6.3 术语数据规范
- `term`：术语名称，唯一
- `definition`：术语解释，详细明了
- `category`：术语分类，如"项目"、"编程"、"人物"
